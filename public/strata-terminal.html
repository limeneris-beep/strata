How Verification Works

Zero-Trust Architecture: Your browser verifies the engine
1
Cryptographic Snapshot

Every few seconds, the backend engine compiles every active order into a Merkle tree and produces a single 32-byte hash (the Merkle Root).

2
On-Chain Commitment

The engine signs the root and submits it as a Solana transaction to the permanent Memo program. This commits the engine to that exact state.

3
Automated Client Verification

Your browser automatically downloads the raw order book data, independently recomputes the Merkle root using SHA-256, and verifies it matches the Solana blockchain.
